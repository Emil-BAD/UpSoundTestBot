## 4
Добавь в существующий телеграм бот сервис для получения информации о треке из Яндекс Музыки по ссылке.
!!Это не отдельный скрипт, а часть проекта. Реализация должна быть в слое services.

Задача:
пользователь отправляет ссылку на трек из Яндекс Музыки, а бот возвращает:
- название трека
- исполнителя
- длительность
Сделай реализацию с учётом следующих требований:
Архитектура:
вся логика парсинга должна находиться в services 
хэндлеры только принимают ссылку и вызывают сервис
никакой бизнес-логики в хэндлерах

Реализация:
Сервис - функция или класс, который принимает ссылку на трек Яндекс Музыки. Валидирует ссылку (что это именно трек). Делает HTTP-запрос к странице. Парсит данные (название, артист, длительность). Возвращает результат в удобном виде

Парсинг - используй requests + BeautifulSoup (или предложи более надёжный способ, если есть лучше). Обработай возможные ошибки:
 невалидная ссылка
  трек не найден
  изменения структуры страницы
  отсутствие данных

Обработка ошибок - сервис не должен падать, возвращай понятные ошибки. Можно сделать кастомные исключения

Интеграция с ботом - добавь хэндлер, который: принимает сообщение с ссылкой, вызывает сервис, красиво форматирует ответ пользователю (на русском языке)

Код:
чистый и читаемый
легко расширяемый 
без хардкода
с комментариями, без перегруза

Дополнительно:
если Яндекс Музыка сложно парсится, предложи альтернативу
Строго соблюдай разделение на handlers и services

## 10
В проекте используется прокси, который указан в .env, и он применяется глобально для всех сетевых запросов.

Проблемы:
telegram бот работает через прокси (и это правильно)
но запросы к внешнему API (в частности yandex-music API через библиотеку yandex-music) не работают через этот прокси
из-за этого API возвращает ошибки или недоступность

Использую десктопный VPN, прокси нужен только для тг (чтобы бот был доступен в сети), но НЕ должен использоваться для HTTP-запросов к сторонним API.

Задача:

1. Разделить сетевую конфигурацию:
   - Telegram Bot должен работать через прокси из .env (если он указан)
   - все внешние HTTP-запросы (requests, yandex-music, API сервисы) должны выполняться БЕЗ прокси
